# プロジェクト管理の基本設定

## 1. Gitによるバージョン管理の設定

### 初期設定
```bash
# Gitリポジトリの初期化
git init

# 初回コミット
git add .
git commit -m "初期設定：プロジェクトの基本構成"
```

### コミットメッセージの規則
以下の形式でコミットメッセージを記述することを推奨します：

- 新機能追加：「追加：○○機能の実装」
- 修正：「修正：○○の不具合対応」
- 更新：「更新：○○の内容変更」
- 削除：「削除：不要な○○を削除」

## 2. バックアップの自動保存設定

### Cursorの設定手順
1. `File` → `Preferences` → `Settings`を開く
2. 以下の項目を設定：
   - `Auto Save`：`afterDelay`
   - `Auto Save Delay`：30000（30秒）
   - `Files: Auto Save`：`onFocusChange`

### バックアップポイントの作成
重要な更新時には、以下の形式でコミットを作成：

```bash
git add .
git commit -m "バックアップ：○○の実装完了（yyyy年mm月dd日）"
```

## 3. 復元方法

### 特定のバージョンに戻す場合
```bash
# コミット履歴の確認
git log --oneline

# 特定のバージョンに戻る
git checkout [コミットID]
```

### 完全に以前の状態に戻す場合
```bash
# 強制的に特定のバージョンに戻す
git reset --hard [コミットID]
```

## 4. 定期的なバックアップのベストプラクティス

1. 1日の作業開始時にコミット
2. 重要な機能実装完了時にコミット
3. 1日の作業終了時にコミット
4. 週次でのバックアップの確認

## 5. トラブルシューティング

### バックアップが見つからない場合
1. `.git`フォルダの存在確認
2. `git log`でコミット履歴の確認
3. `git reflog`で全ての操作履歴の確認

### 復元できない場合
1. 作業ディレクトリのクリーン状態確認
2. `git status`で現在の状態確認
3. 必要に応じて`git stash`で作業を一時退避

## 6. 注意事項

- 重要な変更前には必ずバックアップを作成
- `.gitignore`で不要なファイルを除外
- 大きなバイナリファイルは別途管理
- 定期的なリモートリポジトリへのプッシュを推奨

## 7. 補足：バックアップ情報の管理

プロジェクトルートに`backup-info.json`を作成し、以下の形式で管理：

```json
{
  "latest_number": 1,
  "backups": [
    {
      "number": 1,
      "date": "2024-03-07",
      "description": "プロジェクトの初期設定完了"
    }
  ]
}
```
